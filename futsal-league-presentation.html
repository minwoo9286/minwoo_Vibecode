지역 풋살 동호회를 위한 분기별 시즌제 토너먼트 운영 웹 서비스

"우리 동호회가 시즌 챔피언이 되는 곳"
Problem · 문제 정의

동호회 운영진은 매주 이런 일에
시달리고 있다

  • ✕ 상대팀을 카카오톡·인맥으로 일일이 섭외해야 한다
  • ✕ 대진표와 일정을 매번 엑셀로 수작업한다
  • ✕ 경기 결과 스코어 시비로 단톡방 분쟁이 발생한다
  • ✕ 시즌이 끝나면 우승 기록이 흔적도 없이 사라진다
  • ✕ 운영진 한 명에게 모든 행정 부담이 쏠린다
Differentiation · 차별화

플랩풋볼과는 다른 영역을 공략한다

항목플랩풋볼본 프로젝트

구장플랩 제휴 구장만동호회 자율 선정
운영플랩 매니저 파견동호회 자율 운영
기록개인 레벨 점수팀 전적·우승 히스토리
결과 입력매니저 단독양 팀 상호 입력·교차 검증
User Scenario · 핵심 시나리오

등록부터 우승까지, 자동으로 흐른다

1 정상 플로우

  • 강남FC 대표가 가입 후 팀 등록
  • "2026 봄 서울 풋살컵" 참가 신청
  • 8팀 확정 → 대진표 자동 생성
  • 경기 후 양 팀 스코어 입력 → 일치 → 확정
  • 다음 라운드 자동 매칭 → 결승 → 우승
  • 명예의 전당에 영구 기록

2 분쟁 해결

  • 두 팀 입력이 일치할 때까지 재입력
  • 한 팀이 48시간 미입력 시 자동 몰수승
  • 운영자 개입 0건을 목표로 설계
  • 외부 단톡방 분쟁을 시스템 내부로 흡수

Key Decision · ADR-001

경기 결과 입력 방식

양 팀 대표가 각자 스코어를 입력하고, 일치하면 자동 확정한다. 운영자 단독 입력은 부담이 크고, 한 팀만 입력하면 결과 조작이 가능하다. 양 팀 상호 입력은 분쟁을 시스템적으로 차단한다.

SCHEDULED
경기 예정
AWAITING
결과 입력 대기
CONFIRMED
양 팀 일치 → 확정
FORFEITED
미입력 → 몰수승
